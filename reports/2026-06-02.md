## 6. Build Signals for K Platform

### 🥇 Adversarial robustness hardening for K pathology foundation models
**Rationale:** The new "universal and transferable attacks on pathology foundation models" paper turns adversarial robustness from theoretical risk into a procurement-blocker question. AstraZeneca/Sanofi security review boards will ask about it within weeks. We need a robustness benchmark, mitigation (input sanitization + adversarial training on tiles), and a public posture before competitors.
**Effort:** Medium (6–8 weeks; one ML eng + sec review)

### 🥈 K MCP server for pathology + genomics tools
**Rationale:** MCP is consolidating as the agentic tool standard (MCP-Persona benchmark, Anthropic momentum, Benchling-style ELN integration paths). Exposing K's signatures, biomarker discovery, and slide-search as MCP tools makes K the default biomedical backend for any pharma's chosen agent framework — a defensive distribution play against the OpenAI-on-AWS direct path.
**Effort:** Small–Medium (4–6 weeks; one backend eng + DevRel)

### 🥉 Per-tenant PEFT adapter architecture for pharma data residency
**Rationale:** The PEFT-at-scale paper validates a "one base K model + many partner adapters" architecture that solves three problems at once: pharma data isolation, GxP traceability, and cost. It also creates a long-term lock-in surface (each partner's adapter is a switching cost). Aligns with OpenAI/AWS competitive pressure — we offer something they architecturally can't.
**Effort:** Large (one quarter; ML infra + platform team)
