## 6. Build Signals for K Platform

### 🥇 Kinase Inhibitor–Mutation Response Map Integration
**Rationale:** The Nature Biotech kinase profiling dataset is the most immediately actionable pharma-relevant dataset surfaced today. Integrating mutation-specific inhibitor sensitivity maps into K Platform's biomarker module would directly serve AstraZeneca and Sanofi's oncology pipelines, enable drug repurposing workflows, and differentiate against Benchling/Biomni which lack this layer.
**Effort:** Medium (2–3 sprints) — requires data ingestion pipeline, mapping to existing compound/target ontologies, and UI for exploratory queries.

### 🥈 Declarative Agent Workflow Specification (AgentSPEX-Compatible)
**Rationale:** As K Platform moves toward agentic pipeline orchestration, adopting or aligning with an emerging agent specification standard (like AgentSPEX) would future-proof interoperability with Anthropic MCP, third-party ELN tools, and enterprise automation platforms. OpenAI's Codex enterprise push makes this urgent — pharma will expect composable agent architectures.
**Effort:** High (3–4 sprints) — architectural decision + refactoring of current pipeline orchestration layer.

### 🥉 Retroactive Data Deletion / Provenance Audit Trail
**Rationale:** The PubMed paper on patient-initiated EHR data deletion surfaces a real compliance gap. Under EU AI Act Article 15 (traceability) and HIPAA, K Platform must be able to demonstrate which training data was used for which model and support retroactive removal. This is table-stakes for enterprise pharma procurement and will come up in GxP validation audits.
**Effort:** Medium-High (2–3 sprints) — requires data lineage metadata store, model-data dependency graph, and deletion propagation logic.
