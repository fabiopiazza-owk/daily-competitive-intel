## 6. Build Signals for K Platform

### 🥇 Model-agnostic orchestration layer for oncology (K-Orchestrator)
**Rationale:** LCA ([arXiv](http://arxiv.org/abs/2607.06531v1)) crystallizes what pharma buyers want — decoupled ingestion, clinical routing, and model inference with pluggable backbones (K's own + Gemma 4 + third-party). Without this, K risks being seen as monolithic against Biomni/Benchling's more composable stories. Ship a reference architecture doc + MCP-compatible API in Q3.
**Effort:** Large — 1 quarter for MVP, cross-team (platform, oncology models, MCP integration).

### 🥈 LLM-powered pharmacovigilance/AE causality module
**Rationale:** [WHO-UMC LLM causality paper](https://pubmed.ncbi.nlm.nih.gov/42418253/) shows this is a tractable, high-ROI workflow. AstraZeneca and Sanofi both have massive PV cost centers; a K module for FAERS/EudraVigilance triage is a strong land-and-expand opportunity beyond biomarker discovery.
**Effort:** Medium — 6–8 weeks for a scoped MVP on FAERS with human-in-the-loop review.

### 🥉 IHC + morphology multimodal fusion reference model (breast/DCIS)
**Rationale:** [TRPV4 IHC + DL DCIS paper](https://pubmed.ncbi.nlm.nih.gov/42417209/) validates K's multimodal pathology thesis with a publishable, buyer-facing case. Package as an AstraZeneca-ready reference deployment demonstrating IHC-panel + H&E fusion; positions K ahead of pure-H&E competitors.
**Effort:** Medium — 4–6 weeks leveraging existing K pathology models plus IHC ingestion adapters.
